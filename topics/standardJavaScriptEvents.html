JavaScript Events

There are a lot of events that are available for JavaScript. These events are simply things that happen to HTML elements either through something the user does or something the browser does. One of the most commmon events is the onclick event that triggers when an element is clicked. When that item is clicked it calls a function that then does whatever the programmer specifies. There are many other user initiated events.

Other events are triggered by the browser like onload. This event is triggered when the object is loaded by the browser. The most common use is to put onload inside of the <body> element to cause a JavaScript function to run once the page is loaded. Wherever it appears it will run JavaScript once that element is finished loading. On this page, for example, there was an alert() that was triggered once the page finished loading due to a onload event in the <body> element of the page.

Mobile Friendly Events

There are some events that are more mobile friendly than others. Some of these events are touch events like touchstart, touchmove, touchend, and touchcancel. They work as follows:

  • touchstart - triggers when the element is touched by the user
  • touchmove - triggers when the user touches the element and then moves his or her finger across the screen
  • touchend - triggers when the user stops touching the element and the screen (not to be confused with touchmove which still works even if the user isn't touching the element as long as he or she is still touching the screen)
  • touchcancel - triggers when a touch is interrupted by something else. This seems to be used as an error handling type event when the normal sequence of touchstart, touchmove, and touchend is interrupted for some reason like too many touch points, pressing the home button while touching, and/or pressing the lock button while touching. A great article outlining when this is actually used can be found here.
