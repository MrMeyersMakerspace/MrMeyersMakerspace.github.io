HTML5 Tags - Audio, Video, and Canvas

HTML5 added a ton of new features for web developers to use including more forms of input, input attributes, form elements, structure elements, graphic elements, and media. This page is going to focus on three of those new tags, the <audio>, <video>, and <canvas> tags. These allow you to easily embed natively hosted audio and video content as well as create graphics on the fly using, usually, JavaScript. Before these tags all of these features were only accomplished by using plugins, and couldn't be handled with native HTML. Now we can do it with straight HTML. We'll dive into each individually, what they bring to the table, and how they work.

Audio Tag

The audio tag allows you to embed audio into your site without having to use an external plugin. You have control on whether or not you have controls available for the user and whether or not that audio automatically plays or is trigged by the user. These controls can even been controlled via JavaScript so you can create your own control buttons instead of the build in controls. All browsers support MP3 files, with most also supporting WAV files, and all but Edge, IE, and Safari also supporting OGG files.

To create an audio element we simply use the <audio> tag, define the source, and then what type of audio file it is. Then you can add controls if you want the audio to also have controls available for playing, pausing, controlling the sound level, and the ability to download the audio file. Below is a sample music file with the code necessary for it to work. Not shown is an audio file called "music.mp3" that is also hosted in the same folder.


Code:

<audio controls src="music.mp3" type="audio/mpeg">Your browser does not support the audio element</audio>

You'll notice the text inbetween the <audio> tags. This text will appear if the user is running the site on an older browser that does not support the <audio> tag or the audio file type used. This is also true for the <video> tag as well.



Video Tag

The video tag, like the audio tag, allows you to embed media in your page without the need of an external plugin, in this case video. Also like the audio tag you can decide whether or not the video will have controls built in and whether or not that video automatically plays. These controls can also be controlled externally via JavaScript. Video also allows for fullscreen and picture-in-picture viewing with the embedded controls. Another feature of video is you can control how big the video is by defining the width and length of the video. If you define just one it will scale the other appropriately. If you define both but don't use the same ratio as the original video then it will create a video element of that size but keep the proper ratio on the video as it is playing.

Below is an example video that has embedded controls with it's code. The volume control doesn't work because this stock video has no audio.


Code:

<video controls height="300" src="beachVideo.mp4" type="video/mpeg">Your browser does not support the video element</video>

For a video with external controls we can remove the controls statement in the <video> tag and then create JavaScript controlled buttons to do the same thing, just like with audio. You can also change the size of the video as well by altering the height or width values via JavaScript. Here is an example of a video with no onboard controls that is controlled with external buttons instead:

Video Code:

<video height="300" src="beachVideo.mp4" type="video/mpeg" id="myVid">Your browser does not support the video element</video>

Code for Buttons:

var vid = document.getElementById("myVid");

// Play/pause button for video
function playPauseVid() {
    if (vid.paused)
        vid.play();
    else
        vid.pause();
}

// Small button for video
function small() {
    vid.height = 200;
}

// Original button for video
function original() {
    vid.height = 300;
}

// Big button for video
function big() {
    vid.height = 400;
}

Other handy features for video are autoplay, loop, and muted. You can use autoplay to have a video automatically start playing when a page is visited. Loop will have that video (also works with audio) continue to loop indefinitely or until stopped by the user (as long as the controls are present). Muted will have the video play but mutes the audio. If you want a video to autoplay you must also have the muted property as well or most modern browsers will block the autoplay. All you have to do to add these features is to add autoplay, loop, and/or muted respectively to the tag properties. Below is a video that automatically plays when the page is loaded and continues to loop without the ability to stop it because the controls are disabled. Be careful on how you do this as it could irritate users.

Code:

<video height="300" src="beachVideo.mp4" type="video/mpeg" autoplay muted loop>Your browser does not support the video element</video>
